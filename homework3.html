Accessibility Checklist

Website Page Titles

Page titles are the name of the page displayed in the browser bar. It is created with a basic navigable code that allows users to know where they are, keep track of information within specific pages, and often differentiates from other uses of title elements.

Results

Across ten pages, there were a few errors with page titling, codewise and in appearance in the browser tab. Accessible page titles across the website would reflect the specific nature of the page, followed by the name of the organization. On this website, there are some inconsistent titling across pages; such as if it will include the company name and page subject, or only one of the items. On a positive note, there are metadata tags which add an extra description of the content to be found within each page based on its < title >. These tags are useful in the event of updates and remodeling of certain code elements where people can keep track of their progress.

Image Alternative Text

Alternative Text is used as a descriptor under images and other like items. The feature of alternative text allows users to understand the context of a certain element that they may not otherwise see without it. For screen readers specifically, this feature allows them to represent an image in a way for the user to understand. It is also a matter of describing the functions of links within a page when there is slow internet connection or an error in the src attribute.

Results

There are many images and icons across the website that do not have alternative text attributes to the code. This also causes the links of the images to be empty, another issue with trying to find accessible information on the image or other link. One page with many hits for empty, or broken links and no alt text applications is Publications. For the amount of logos and images across the page, there are few supportive text descriptions. Another area where this can be seen is in the staff page, where there is no alt text provided for the images of each staff member which are a two picture moving piece. While the staff descriptions are below the card/ within the attached link, using alt text on the images would be supportive to those who need more context on what an image shows, if they cannot see it easily or without assistance.

To add in alternative text elements, the code may look like this: < img src="____" alt="____"> . This is the simplest snippet for adding alternative text and it can be applied to many different image elements (.gif, .jpg, .svg, etc.). Descriptions do not have to be incredibly detailed either, however should provide an idea of what the image displays.

Text

  • Headings

    • The use of headings across a website allows for all content to be organized within specific areas and topics, separated with the use of other headings. Headings are labeled in order of importance: < h1; h2; h3; > which designates the following information into sets and subsets based on importance. Having headings is an important part of accessibility because it provides structure, and easier navigation when using assistive technologies.

    Results

    After reading through the use of code across 10 pages, all headings were identified and headings were ordered correctly based on the information organization. It is clear to see where content belongs and separate various sections. However, there are many ways to advance the organization and make more clearly designated spaces both within the code and for the layout of the website. Many of the pages which list the various policy successes across the state are sectioned off in a way that is not supportive to assistive technologies and the ability to separate information clearly. For example, on the page Youth Mental and Behavioral Health, the use of paragraph < p > tags and < span > tags create a dense section of organization which 'flows' rather than identifies separate entities of information for the assistive reader or user. Instead, consider the use of bulleted list items which are a simple way to break up information to be more digestible, but still maintain consistent patterns in formatting and display.

  • Contrast Ratio

    • Contrast Ratio refers to the color schemes used to provide a clearly difference between text and background colors. Having high contrast between colors and sizes used for text, and the background colors provides a more accessible setup for users, especially those who have low vision. Without high contrast, users may struggle to identify text across the webpage or find other navigable features.

    Results

    Across the website, color contrasts received a low score for accessibilty. The background colors and text are listed as incompatible because of very low color contrast and this affects multiple pages. The code areas all denote that the contrast setting is on "auto" which does not give a clear indication what ratio is used. The WAI system, however, states that there is less than a 4.5:1 ratio range which is necessary for all text lower than 14 point bold.Listed ratios found across the site include 1:1, 2.82:1, 3.94:1, and 4.19:1. Adjustments for the contrast section of the accessibility checker allows a visual for how changes will appear if utilized in the website code.

    On the Accessibilty Statement page, there is acknowledgment of the known contrast issues and other pieces of the website that need to better adhere to W3C and WCAG 2 AA. This is a positive feature to let users know that specific accessibility changes are in progress.

  • Resize Text

    • Text resizing formats can be supportive to a website if the website has a stable structure across its pages. This means that upon text zooming, there is no issue of congestion or overlap among sections, texts, or images. This is often a browser initiated process,and a separate feature from whole-page zooming which may require an extension to the browser. Having the ability to zoom text size provides users with visual impairments the preferential control to focus on written content only and in a larger size than the standard page.

    Results

    Using the Chrome browser to complete this review, I noticed that the only way to get a text-only zoom feature is through adding an extension to the browser. Once I downloaded the extension, I was able to enhance text on the CO Kids website without any overlap across sections. What I did notice was the images would cut off into the end of their section when text was enlarged, but this had no effect on actually reading the text at a larger scale. Another positive element to this is that the text-wrap feature means that horizontal scrolling does not appear on the page whereas in a whole page zoom, the text will spread across the page and need to be scrolled over to complete.

Interactions

Keyboard Access and Visual Focus

To provide a higher level of accessibility for users with impairments, keyboard access and visual focus refers to the ability to use alternative methods of movement around a webpage.

  • Tab to all: Using only the tab button and arrows, all pages are accessible including links and form fields.
  • Tab away: It was not as easy to get out of sections once tabbed in. Using keyboard features, the option was easier going forward rather than retracting out of tab areas.
  • Tab order: The left-to-right, top-to-bottom sequence is followed when tabbing through sections
  • Visual focus: When tabbing through the various sections, it is difficult to see what is outlined, but there is a faint definition to the selected section. A way to enhance this is by adding more contrast to the focus pieces, whether that is adding an inverted background, or using a shadow effect to signify that something is clearly selected.
  • All functionality by keyboard: There are some inconsistent areas across the entire website in the ability for it to be accessed using keyboard functions only. When trying to select elements on certain pages, such as a search page, the tab feature and arrow features skip across content and into places like the footer.
  • Drop-down lists: The tab and arrow buttons are easy to use for entering a drop-down list and selecting certain sections.
  • Images: There is no issue when using tab and enter buttons for images with links.

Skip to Main

Having a "skip to main" feature allows users to bypass content that is not applicable to their needs.It also supports a streamlined selection process for users who may use assistive devices.

Results

From the homepage, navigation for the "Skip to main" also includes "skip to primary content," "skip to primary sidebar," and "skip to footer." Upon using the tab method of navigation, these options become highlighted along with a "search" option that drops from the search toggle bar. This is a great way for users to find what they need quickly and without clutter to their queries. An even more useful feature is the global header and footer which means the "skip to" feature can be accessed on any page.

Forms, Labels, and Errors

Fill-in forms, such as this Stay Informed Sign-Up form should have clearly designated rules for how to fill each box so that an assistive device would know how to direct its user. While it is a third party website, it can still be toggled in some ways via the organization to ensure accessible features.

Results

I ran this external link through the WAVE to measure its accessibility and while it uses ARIA features which are semantic-enhancing for websites, there were issues in contrast and with the presence of broken links. The form itself uses a fieldset and legend which organizes and categorizes the elements so they appear uniform and allows for screen readers to relay the information back to the user. Enlarging the text and providing higher contrast or bolding the sections of the form may also be more supportive to users with impairments.

General

Moving content

Moving content can be a testy feature to websites when regarding their accessibility. How fast is the section moving? What combination of colors, text, and images, are present? What type of moving content is it? These are all questions that must be considered when going forth with a moving section. For some people with visual impairments, processing disorders, or even epilepsy, too much movement, and/or flash-heavy imagery can be incredibly harmful. Stylistic choices such as a slow rotation for moving elements also supports people who may read at a slower pace.

Results

The moving carousel on the homepage of the CO Kids website is coded with delays in the movement and animation process and only features three slides. The images and text used on this carousel are limited and large in size; avoiding too much in an already moving piece. While there is an auto-animation for the slides, they do not start right away and two cursors on either side of the carousel allow for the user to toggle left or right at their own pace. The animation is coded as "Ease Out" and automatically loops after six seconds.The carousel also is reserved to the top so vertical scrolling does not pull down or cause clutter with continuous moving.

Based on recommendations for accessibility by the WAI, the carousel does not run the risk of content flashes and blinks at more than three times in one second. The one area that could be enhanced is the ability to completely stop the movement of the carousel when it is in view. By clicking on one of the slides it automatically redirects the user to a new page, instead of pausing or holding the slide still.

Multimedia Alternatives

Multimedia alternatives provide heightened accessibility for users with choices that can be customized as needed. This includes but is not limited to, closed captioning services, volume toggles, resolution enhancement, and visual format support for screen readers.

Results

I found no multimedia features on the CO Kids website.
